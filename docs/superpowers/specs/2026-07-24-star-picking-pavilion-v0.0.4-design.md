# 摘星阁 v0.0.4 桌面后台运行设计

**状态：** 已确认
**目标版本：** `v0.0.4`
**日期：** 2026-07-24

## 1. 背景与目标

v0.0.3 已解决长期运行时的数据库膨胀、周期性聚类写放大、热点查询开销和失效信源空转，并通过 219 个单元测试、真实 Electron 回归和生产依赖审计。当前最明显的产品缺口不在采集算法，而在桌面生命周期：应用定位为持续工作的情报站，但关闭最后一个窗口会直接结束 Electron 和后端服务，用户无法选择让它安静驻留。

v0.0.4 将补齐可控的 Windows 后台运行能力：

- 用户可在设置中独立开启或关闭“关闭窗口后在后台运行”。
- 用户可在设置中独立开启或关闭“登录 Windows 时自动启动”。
- 两项设置默认均为关闭，升级后保持 v0.0.3 的既有关闭习惯。
- 后台运行开启后提供系统托盘入口、明确退出路径和首次隐藏提示。
- 同时开启两项设置时，Windows 登录启动不弹出主窗口，应用直接进入后台。
- 不削弱现有的单实例、安全隔离、自动更新和后端优雅关闭机制。

## 2. 方案比较

### 方案 A：用户可控的完整后台模式

同时实现关闭转托盘和 Windows 登录启动，两项独立设置、默认关闭。该方案完整覆盖长期驻留场景，同时避免升级后强制改变用户习惯，采用此方案。

### 方案 B：只实现关闭转托盘

改动较小，但用户重启电脑后仍需手动打开应用，无法形成连续情报采集链路，不足以补齐长期驻留能力。

### 方案 C：默认强制后台运行

连续运行最直接，但关闭按钮语义发生未经选择的变化，容易造成“应用无法退出”或后台资源占用的误解，不采用。

## 3. 范围与非目标

### 3.1 本版本范围

- Electron 后台生命周期控制器。
- 按需创建和销毁的系统托盘。
- 托盘菜单中的“打开摘星阁”和“退出摘星阁”。
- 关闭主窗口时按设置决定隐藏或退出。
- Windows 登录启动开关与实际系统状态回读。
- 登录启动时根据后台设置决定显示或隐藏主窗口。
- 设置页中的桌面运行卡片、状态、错误提示和可访问开关。
- preload 与 IPC 的最小桌面设置接口。
- 设置持久化、单实例唤醒、托盘退出和真实 Electron 回归。
- v0.0.4 版本、文档、安装包和 Release 元数据。

### 3.2 非目标

- 不在本版本加入系统通知订阅、热点推送或日报弹窗。
- 不增加 macOS、Linux 或移动端支持。
- 不改变采集、AI 评分、聚类、保留策略和信源退避算法。
- 不把开机启动伪装为应用内布尔值；Windows 的实际注册状态始终是权威来源。
- 不强制任何用户启用后台运行或开机启动。

## 4. 用户体验

设置页新增“桌面运行”卡片，包含两个独立开关：

1. **关闭主窗口后在后台运行**
   - 默认关闭。
   - 开启后，点击窗口关闭按钮只隐藏窗口，后端继续采集和分析。
   - 当前进程第一次隐藏时显示原生提示：“摘星阁仍在后台运行，可从系统托盘重新打开或彻底退出。”
   - 关闭后恢复“关闭窗口即退出”的既有行为。

2. **登录 Windows 时自动启动**
   - 默认关闭。
   - 开启后注册当前安装程序为当前用户的登录启动项。
   - 若后台运行同时开启，登录启动参数包含 `--hidden`，主窗口不弹出。
   - 若后台运行关闭，登录启动后正常显示主窗口。

两项开关立即生效。界面保存状态必须来自 Electron 返回的确认快照，不做乐观伪成功。失败时保留实际状态并显示可操作的错误信息。

后台运行开启时出现摘星阁托盘图标。单击或双击托盘图标均可恢复主窗口；右键菜单提供：

- 打开摘星阁
- 退出摘星阁

“退出摘星阁”必须经过现有的后端协作式关闭流程。第二次启动应用时仍只保留一个实例，并恢复、显示和聚焦已有窗口。

## 5. 状态与持久化

### 5.1 后台运行偏好

`closeToTray` 加入现有版本化 `ui-preferences.json` 白名单，类型为布尔值，默认 `false`。旧文件缺少该字段时按默认值补齐，不提升偏好文件版本，不需要破坏性迁移。

该值控制：

- 是否创建托盘；
- 用户关闭窗口时是否拦截并隐藏；
- 登录启动参数是否包含 `--hidden`。

### 5.2 开机启动状态

`launchAtLogin` 不写入 JSON。Electron 主进程使用：

- `app.getLoginItemSettings()` 读取 Windows 当前真实状态；
- `app.setLoginItemSettings()` 修改当前用户登录启动项；
- 修改后再次读取并校验结果。

应用只在已打包的 Windows 运行时允许修改登录启动项。开发模式返回 `supported: false`，设置界面禁用该开关并说明只有安装版可配置。单元测试通过注入的 Electron 适配器验证全部分支。

### 5.3 登录启动参数同步

当 `launchAtLogin` 已开启而 `closeToTray` 发生变化时，主进程同步重写登录启动参数：

- `closeToTray: true`：加入 `--hidden`；
- `closeToTray: false`：移除 `--hidden`。

同步失败不得影响当前窗口的后台运行设置，但必须返回警告并展示给用户，避免隐藏失败。

## 6. 架构与组件

### 6.1 后台生命周期控制器

新增独立 Electron 模块，负责：

- 持有当前 `closeToTray` 运行状态；
- 按需创建、更新和销毁 `Tray`；
- 统一执行窗口显示、恢复、聚焦和隐藏；
- 判断窗口关闭事件应隐藏还是继续退出；
- 记录当前进程是否已经展示过后台提示；
- 查询和设置 Windows 登录启动；
- 在偏好变化后同步登录启动参数；
- 在应用退出时销毁托盘并停止接管关闭事件。

控制器通过依赖注入接收 `app`、`Tray`、`Menu`、`Notification`、窗口访问器和图标路径，单元测试不启动真实 Electron。

`electron/main.js` 只负责组装控制器和转发生命周期事件，不继续堆叠托盘细节。

### 6.2 IPC 与 preload

新增两个最小 IPC 通道：

- `desktop-settings:get`：返回确认快照；
- `desktop-settings:update`：只接受已知字段的部分更新。

快照结构：

```js
{
  closeToTray: false,
  launchAtLogin: false,
  launchAtLoginSupported: true,
  warnings: []
}
```

更新请求只允许：

```js
{
  closeToTray?: boolean,
  launchAtLogin?: boolean
}
```

未知字段、非普通对象和非布尔值同步拒绝。preload 暴露深冻结的 `getDesktopSettings()` 与 `updateDesktopSettings(patch)`，不向渲染层暴露 Electron 对象或任意 IPC。

### 6.3 渲染层

桌面设置控制逻辑从 `renderer/app.js` 的通用页面装配中分离为小型控制器，负责：

- 加载确认快照；
- 渲染两个开关及支持状态；
- 串行提交变更，避免快速点击造成顺序反转；
- 提交期间禁用相关控件；
- 使用返回快照更新界面；
- 显示错误或部分成功警告；
- 在开发模式禁用登录启动开关。

样式沿用现有玻璃卡片、按钮、状态文本和深浅主题，不进行无关的整体视觉重做。

## 7. 生命周期与数据流

### 7.1 正常启动

1. Electron 完成单实例锁、安全策略和用户数据迁移。
2. 加载 `ui-preferences.json`。
3. 根据 `closeToTray` 初始化后台控制器和托盘。
4. 启动后端并创建主窗口。
5. 只有命令行含 `--hidden`、`closeToTray` 为真且托盘创建成功时，主窗口保持隐藏；否则正常显示。

该三重条件防止损坏偏好或托盘初始化失败把应用困在不可见状态。

### 7.2 用户关闭窗口

1. 若应用正在执行真正退出、更新安装或后台运行未启用，允许既有关闭流程继续。
2. 若后台运行已启用且托盘可用，阻止窗口销毁并隐藏。
3. 当前进程第一次隐藏时显示原生后台提示。

### 7.3 托盘恢复与第二实例

托盘激活、托盘菜单“打开摘星阁”和第二实例事件统一调用同一 `showMainWindow()`：

- 最小化时先恢复；
- 隐藏时显示；
- 最后聚焦。

### 7.4 真正退出

托盘菜单“退出摘星阁”、关闭按钮的正常退出以及自动更新安装继续复用现有 `before-quit` 门禁：

1. 标记正在退出，停止后台控制器接管窗口关闭。
2. 停止更新定时器。
3. 请求后端关闭 HTTP 服务、调度器和 SQLite。
4. 后端确认或超时后结束 Electron。

## 8. 错误处理

- 托盘创建失败：记录固定格式错误，保持主窗口可见，关闭按钮继续真正退出，设置页报告后台运行不可用。
- 后台偏好写入失败：恢复控制器的上一个确认状态并返回错误。
- 登录启动写入失败或回读不一致：保留 Windows 实际状态，返回明确错误。
- 已开启登录启动时同步 `--hidden` 参数失败：后台设置仍可生效，但返回警告。
- 原生通知不可用或展示失败：静默降级，不影响隐藏和托盘恢复。
- 托盘图标或窗口已销毁：控制器方法幂等，不重复销毁或操作失效对象。
- 启动时检测到 `--hidden` 但后台模式不可用：忽略隐藏参数并显示主窗口。

所有返回到界面的错误使用固定、可理解的中文消息，不透传任意系统异常文本到 HTML。

## 9. 测试策略

严格采用测试先行：

### 9.1 单元测试

- 后台设置默认关闭，旧偏好文件可无损补齐。
- 偏好白名单拒绝未知字段和非布尔值。
- 开启后台运行时创建托盘；关闭时销毁托盘。
- 关闭窗口在启用时隐藏，在禁用或退出中时真正关闭。
- 首次隐藏提示在单个进程中最多出现一次。
- 托盘激活、菜单打开和第二实例复用统一显示逻辑。
- 托盘退出进入现有优雅关闭路径。
- 开机启动只在打包 Windows 上可修改。
- `setLoginItemSettings()` 后回读实际状态。
- 后台设置变化会同步 `--hidden` 参数。
- 托盘失败、偏好失败和登录启动失败均回退到安全可见状态。
- IPC 只接受白名单布尔字段。
- preload API 深冻结且不扩大权限边界。
- 渲染控制器串行更新、按确认快照渲染并处理警告。

### 9.2 集成与真实 Electron 测试

- 在真实 Electron 中开启后台运行后关闭窗口，窗口隐藏但进程与后端保持存活。
- 启动第二实例后，已有隐藏窗口恢复并聚焦。
- 通过明确退出路径后，后端在现有超时门限内协作关闭。
- 使用测试数据目录冷启动，确认 `closeToTray` 跨重启恢复。
- 使用 `--hidden` 且后台模式有效时保持隐藏；后台模式无效时仍显示窗口。

## 11. 验收标准

- 升级用户不改设置时，关闭窗口行为与 v0.0.3 一致。
- 用户开启后台运行后，关闭窗口不会停止采集，托盘可以恢复或彻底退出。
- 用户可独立控制 Windows 登录启动。
- 同时开启两项设置后，登录启动不弹出主窗口。
- 任一托盘或启动项故障都不会留下无窗口、无托盘但仍运行的不可见进程。
- 单实例、自动更新、凭据加密、随机回环端口和优雅关闭回归保持通过。
- v0.0.4 安装包通过完整发布门禁并由公开 Release 提供。
