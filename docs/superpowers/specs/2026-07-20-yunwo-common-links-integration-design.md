# 摘星阁合并云幄常用网址功能设计

## 目标

将“捕风司”的用户可见产品名称统一改为“摘星阁”，保持摘星阁现有情报站架构、信息流、日报、信源和设置功能不变；只把“云幄”当前的常用网址功能完整合并为摘星阁中的一个原生页面。

## 范围

### 包含

- 在摘星阁顶部主导航中新增“常用网址”标签页。
- 页面标题使用“云幄 · 常用网址”，保留“云幄”名称和功能来源身份。
- 原样迁入云幄当前 14 个网址的名称、URL、分类、说明、标签和默认星标状态。
- 提供全部分类和六个业务分类的筛选。
- 支持切换星标、星标本地持久化和星标优先稳定排序。
- 使用摘星阁现有深空夜航与宣纸白主题、玻璃卡片、字体和响应式布局。
- 通过现有 Electron 外链拦截机制在系统浏览器中打开网址。
- 将用户可见和内容身份中的“捕风司”改为“摘星阁”。

### 不包含

- 不迁入云幄的 AI 工作台、项目台账、会议纪要、资料库、待办协同、更新服务或其他模块。
- 不增加云幄当前常用网址源码中不存在的新增、编辑、删除、拖拽、访问统计或搜索能力。
- 不修改云幄源码。
- 不改动 `windcatcher` 包名、数据库文件名、预加载桥名称、应用 ID、GitHub 仓库地址和更新标识；这些内部标识继续用于兼容已有数据及自动更新。
- 不调整摘星阁现有情报业务接口和数据库结构。

## 方案选择

采用“原生融入”方案：将常用网址的数据和纯业务规则放入独立、可测试的原生 JavaScript 模块，在摘星阁现有 HTML、JavaScript 和 CSS 中增加一个视图并接入原导航。

未采用以下方案：

- 嵌入云幄 React/Vite 构建页：会在同一应用维护两套前端运行时、主题和状态系统，增加打包体积与维护成本。
- 打开独立云幄窗口：没有形成摘星阁内部的统一功能与导航，不满足完整合并的要求。

## 架构与组件

### 常用网址领域模块

新增 `renderer/common-links.js`，只负责：

- 暴露 14 个不可变网址条目。
- 生成去重后的分类列表。
- 校验本地保存的星标 ID，只保留当前仍存在的网址。
- 在指定分类下筛选网址。
- 按“星标优先、原始顺序不变”的规则排序。

模块同时支持浏览器全局对象和 CommonJS 导出，以便页面直接使用并由 Node 内置测试加载，不引入新运行时依赖。

### 页面视图

在 `renderer/index.html` 中：

- 主导航新增 `data-view="links"` 的“常用网址”标签。
- 主内容新增 `viewLinks` 视图。
- 视图包含“云幄 · 常用网址”标题、功能说明、分类按钮容器和网址卡片容器。
- 在 `app.js` 前加载 `common-links.js`。

### 页面控制逻辑

在 `renderer/app.js` 中：

- 将 `links` 纳入合法视图并接入现有 `switchView`。
- 首次进入或切换分类时渲染常用网址卡片。
- 星标点击后立即更新集合、写入本地存储并重新稳定排序。
- 所有数据写入 DOM 前继续使用现有 `esc` 转义工具。
- 非信息流视图继续隐藏领域与情报分类筛选条；原有顶部情报搜索行为不变。

### 视觉样式

在 `renderer/styles.css` 中新增带 `common-links-` 前缀的样式，避免与现有 `.card`、`.pill`、`.empty-state` 等规则冲突。样式直接使用摘星阁现有 CSS 变量，不复制云幄视觉系统：

- 分类筛选使用现有圆角胶囊语言。
- 网址以自适应网格展示，卡片在宽屏下自动分栏，窄屏下单列。
- 星标按钮具有默认、悬停、激活和键盘焦点状态。
- 卡片包含分类、标题、说明、标签和“打开”按钮。
- 尊重现有 `prefers-reduced-motion` 处理。

说明：实现与验收以云幄当前源码中的 14 个条目为准，确保“完整合并”。

## 状态与数据流

1. 页面加载 `common-links.js` 并获得完整网址数据。
2. `app.js` 从 `zxg-common-links-favorites` 读取星标数组。
3. 没有已保存数据时，使用云幄条目中的默认 `pinned` 集合。
4. 保存值不是数组或 JSON 损坏时，回退默认集合；数组中的未知 ID 被丢弃。
5. 用户切换分类时，模块先筛选，再按星标优先与原始顺序稳定排序。
6. 用户切换星标时，页面更新集合、保存数组并重新渲染。
7. 用户点击“打开”时，`target="_blank"` 触发现有 Electron `setWindowOpenHandler`，只允许 `http:` 或 `https:` URL 交给系统浏览器，摘星阁窗口不离开当前页面。

## 错误处理与安全

- `localStorage` 不可用、读取异常或写入失败时，当前会话仍使用内存状态，页面功能不中断。
- 保存数据损坏时恢复云幄默认星标，不显示错误弹窗打断用户。
- URL、名称、分类、说明和标签在拼接 HTML 前统一转义。
- 所有迁入 URL 必须通过 `http:` 或 `https:` 协议校验测试。
- 外链使用 `rel="noopener"`，并复用 Electron 现有导航拦截。
- 不向后端发送网址和星标数据。

## 产品更名

以下用户可见或内容身份改为“摘星阁”：

- HTML 标题和顶部品牌名。
- Electron 窗口标题。
- `package.json` 的 `productName`、描述、作者、构建产品名、版权和快捷方式名称。
- README、发布说明、图标脚本说明、服务启动日志、前端注释与 AI 系统身份文案。

以下内部兼容标识保留：

- npm 包名 `windcatcher` 与 lockfile 对应字段。
- `com.windcatcher.app`。
- `windcatcher.db`。
- `window.windcatcher` 预加载桥。
- `Windcatcher-Setup-*` 产物名、GitHub 仓库与更新配置。

## 测试与验证

### 自动测试

使用 Node 内置 `node:test`，不新增依赖。测试至少覆盖：

- 迁入条目数量、唯一 ID、必填字段和 URL 协议。
- 分类列表与六个业务分类完全一致。
- “全部”和单分类筛选。
- 默认星标集合。
- 星标优先且同组保持原始顺序。
- 保存值缺失、损坏、非数组、空数组和包含未知 ID 时的清洗行为。

### 静态与构建验证

- `npm test` 全部通过。
- `package.json` 和 `package-lock.json` 可解析且依赖树一致。
- 搜索确认用户可见和内容身份中不再残留“捕风司”。
- Electron Builder 配置检查通过。

### 运行时视觉验证

- 启动摘星阁，确认原六个视图仍可切换与加载。
- 确认“常用网址”标签可进入且不会调用情报 API。
- 在深空夜航与宣纸白主题下检查标题、分类、卡片、星标和按钮。
- 检查 1440px、1080px 和 720px 以下宽度布局。
- 切换分类并核对条目数量。
- 切换星标、刷新页面并确认状态与排序保留。
- 点击至少一个 HTTPS 与一个 HTTP 网址，确认系统浏览器打开且摘星阁窗口不跳转。

## 验收标准

- 应用所有用户可见品牌均为“摘星阁”，云幄名称仍以“云幄 · 常用网址”出现。
- 摘星阁现有业务、数据、主题和更新兼容性保持不变。
- 云幄源码中的 14 个常用网址全部可见，内容准确。
- 分类筛选、默认星标、星标切换、持久化、稳定排序与外链打开行为与云幄现有实现一致。
- 未引入云幄其他模块、React/Vite 依赖或后端数据结构变化。
- 自动测试、配置检查、运行时功能检查和视觉检查均通过。
