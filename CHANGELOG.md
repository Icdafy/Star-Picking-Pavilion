# Changelog

本项目遵循 [Semantic Versioning](https://semver.org/)。

## [0.0.5] - 2026-07-25

本版本是一次纯界面版本：重写渲染层设计系统，修复若干长期存在的布局缺陷，并补齐键盘操作路径。数据、采集、AI、凭据与桌面生命周期的行为均未改变。

### Added

- 全键盘操作：`/` 或 `Ctrl+K` 聚焦检索，`Esc` 清空并退出输入，`Alt+1`–`Alt+7` 切换视图，`Alt+T` 换主题，`Alt+R` 立即采集，`Home` 回到顶部；设置页新增快捷键速查卡片。
- 卡片新增「五维研判」显式展开按钮。此前五维分解只能靠点击卡片空白处触发，没有任何可见入口，键盘也无法到达。
- 检索态上下文条：显示当前检索词与已列出条数，并提供一键清除；检索框内新增清除按钮与快捷键提示。
- 长列表新增「回到顶部」浮动按钮，滚动超过一屏后出现。
- 导航新增跟随当前标签滑动的激活指示块；顶部统计数字变化时做短促补间。

### Changed

- 建立设计令牌体系：字号、间距、圆角、动效曲线与语义颜色（accent / second / flag 各含 ink、soft、line 三档）全部令牌化，两套主题只覆写令牌，不再逐条重写规则。
- 字体分层。此前包括按钮、输入框、标签在内的全部界面文字都使用仿宋，小字号下笔画过细。现在控件与元信息改用系统黑体，摘要、情报研判、说明文字仍用仿宋，保留报馆气质的同时提升可读性。
- 卡片头部改为弹性布局，评分胶囊不再依赖标题与元信息上的固定右侧留白，长评分文本不会再压住标题。
- 宣纸白主题下调次要文字色，11–13px 标签在宣纸底上的对比度从 4.07:1 提升到 4.64:1，满足 WCAG AA。
- 全部可聚焦控件补齐统一的 `:focus-visible` 焦点环；标签页与视图建立 `tabpanel` 关联，展开控件补 `aria-expanded`。
- 「加载更多」与「重新生成」在请求期间进入禁用与忙碌态，避免重复提交。
- 页面滚动状态改为在滚动事件内直接判定，不再依赖 `requestAnimationFrame`；窗口隐藏到托盘时帧回调会被暂停，用它会漏更新。

### Fixed

- 修复导航条滚动一屏后跟随内容滑走的问题。`html, body` 的 `height: 100%` 把 `position: sticky` 的包含块钉死在视口高度，导致 sticky 只在首屏生效。
- 修复日期分组标题作为 sticky 元素没有背景、与下方滚动卡片叠字的问题，并让其与右侧热度栏一起按导航条实际高度定位（导航换行时自动跟随）。
- 修复时间轴时间点与竖轴线错位约 6.5px 的问题，两者改为共用同一几何基准。
- 修复实时高亮动画结束后不显式回到常态边框的问题。

## [0.0.4] - 2026-07-24

本版本为长期驻留使用补齐显式的桌面生命周期控制。两个新开关默认关闭，升级后不会自行改变现有启动或退出习惯。

### Added

- 设置页新增「桌面运行」卡片，可独立控制“关闭主窗口后在后台运行”和“登录 Windows 时自动启动”。
- 开启后台运行后，关闭主窗口会隐藏到系统托盘；托盘支持重新打开主窗口和彻底退出摘星阁，首次隐藏时会显示一次系统提示。
- 正式安装的 Windows 版本可注册登录启动；若同时开启后台运行，登录后使用 `--hidden` 静默启动，否则正常显示主窗口。

### Changed

- 第二实例会统一通过桌面后台控制器恢复、显示并聚焦既有窗口。
- `closeToTray` 作为严格布尔值写入版本化 `ui-preferences.json`；登录启动状态始终从 Windows 回读，不重复写入 JSON。
- 安装包白名单新增独立托盘图标资源，并由包审计验证。

### Fixed

- 托盘创建失败时不再进入无恢复入口的隐藏状态，而是安全降级为可见窗口并返回固定提示。
- 桌面设置更新改为串行提交，只渲染主进程确认后的状态；保存失败会回读实际状态，避免开关显示与系统状态不一致。

### Security

- 本版本未改变凭据、网络、AI、数据保留或信源策略。桌面设置仍通过白名单 IPC 暴露，渲染进程无法访问通用 Electron 能力。

### Tests

- 新增后台控制器、托盘生命周期、登录启动回读、IPC 与渲染控制器单元回归。
- 真实 Electron 测试覆盖关闭到托盘、第二实例唤醒、`--hidden` 冷启动、偏好持久化与优雅退出。

## [0.0.3] - 2026-07-24

本版本面向「装上就一直开着」的长期运行场景，解决库体积无限增长、后台周期性卡顿与失效信源空转的问题。

### Added

- 数据保留策略：情报按可配置天数（默认 180 天）保留，判为无关的噪声用更短的独立天数（默认 21 天）；到期条目连同 FTS5 全文索引行一起清除，事件簇随之收敛，清理后截断 WAL。每天日报后 20 分钟自动执行，设置页也可手动触发。
- 设置页新增「数据维护」卡片：显示库内情报数、本地库实际占用空间与待清理条数，并可直接调整两档保留天数。
- 信源失败退避：连续失败的信源按指数拉长重试间隔（一个采集周期起，上限 6 小时），成功即刻复位。信源页显示暂停状态与连续失败次数，并提供「立即重试」；手动「立即采集分析」始终忽略退避，全量重试。
- `GET /api/maintenance` 与 `POST /api/maintenance/prune` 两个本地接口，以及 `POST /api/sources/:id/retry`。

### Changed

- 事件聚类改用倒排索引求交集，替代逐对比较。在真实语料上结果与原逐对实现逐个相同，耗时下降约 5.6 倍。
- 聚类在分组未发生变化时不再写库。此前每 75 秒的分析循环都会重写窗口内全部 `cluster_id`，是 WAL 持续膨胀的主因。
- 热点榜的时间衰减热度改在 SQL 内计算并直接分页，不再每次请求把上千行读进内存后在 JS 里排序。
- `/api/stats` 的全表计数加 5 秒缓存，并在采集、信源变更后立即失效；界面每 18 秒的轮询不再反复全表扫描。
- 新增 `fetched_at`、`COALESCE(published_at, fetched_at)` 与 `(relevant, fetched_at)` 索引，覆盖统计、时间线排序与保留清理的查询路径。

### Fixed

- 修复信息流加载期间切换领域或分类被静默丢弃的问题：整表重载改为以最后一次请求为准，过期响应不再覆盖界面；分页追加仍保持串行。
- 修复检索词中的 `%` 与 `_` 被当作 LIKE 通配符的问题，短词降级检索现在按字面匹配。
- 右侧热度栏同样丢弃过期响应，快速切换领域不会再看到上一个领域的榜单。

### Tests

- 新增倒排索引与逐对比较在多阈值、多领域、含近重复语料下的逐对等价性回归。
- 新增保留清理的 FTS 同步、事件簇收敛与主条改选、日报独立保留期、单轮删除上限等回归。
- 新增失败退避的时长推导、到期判定、健康度分级，以及采集层「失败→跳过→到期重试→成功复位」与强制重试的集成回归。
- 新增热度排序、热度窗口回退、LIKE 转义、数据维护接口与保留天数校验的 HTTP 级回归。

## [0.0.2] - 2026-07-23

### Added

- 使用独立于随机本地端口的 `ui-preferences.json` 记住全部有意义的界面选择：主题、最后视图、领域、动态分类、日报日期、常用网址分类与星标，以及实时更新开关。
- 通过版本化白名单、原子写入和损坏回退，使上述选择在完全退出并重新打开桌面应用后仍能恢复。

### Fixed

- 修复 Electron utility process 的 `MessageEvent.data` 凭据确认未被识别，导致 DeepSeek API Key 虽已加密写入却在界面中显示保存失败、运行时无法调用的问题。
- 将 AI 设置更新、加密凭据保存和普通设置落盘串行为完整事务，避免并发更新失败回滚覆盖较新的成功设置。

### Tests

- 新增两次真实 Electron 冷启动回归测试，强制第二次启动使用不同随机端口，并通过本地模拟 DeepSeek 服务验证加密 Key 恢复和模型调用。
- 新增界面偏好存储、IPC 边界、渲染器恢复、凭据确认及设置事务的单元回归测试。

## [0.0.1] - 2026-07-21

### Added

- Windows x64 桌面安装包、自动更新元数据与本地 SQLite 情报库。
- 低空经济、商业航天信息流、全文检索、聚类、日报与信源管理。
- 可选 AI 五维评分、系统加密凭据存储与“云幄 · 常用网址”视图。

### Security

- 随机回环端口、启动令牌、严格 Host/Origin 校验与请求体限制。
- Electron sandbox、权限默认拒绝、CSP、安全外链与渲染崩溃恢复。
- 原子设置写入、SQLite 完整性检查、旧数据一致性迁移与优雅退出。
- 白名单打包、ASAR 审计、生产依赖审计、SBOM 与 SHA-256 发布门禁。
