# Changelog

本项目遵循 [Semantic Versioning](https://semver.org/)。

## [0.0.3] - 2026-07-24

本版本面向「装上就一直开着」的长期运行场景，解决库体积无限增长、后台周期性卡顿与失效信源空转的问题。

### Added

- 数据保留策略：情报按可配置天数（默认 180 天）保留，判为无关的噪声用更短的独立天数（默认 21 天）；到期条目连同 FTS5 全文索引行一起清除，事件簇随之收敛，清理后截断 WAL。每天日报后 20 分钟自动执行，设置页也可手动触发。
- 设置页新增「数据维护」卡片：显示库内情报数、本地库实际占用空间与待清理条数，并可直接调整两档保留天数。
- 信源失败退避：连续失败的信源按指数拉长重试间隔（一个采集周期起，上限 6 小时），成功即刻复位。信源页显示暂停状态与连续失败次数，并提供「立即重试」；手动「立即采集分析」始终忽略退避，全量重试。
- `GET /api/maintenance` 与 `POST /api/maintenance/prune` 两个本地接口，以及 `POST /api/sources/:id/retry`。

### Changed

- 事件聚类改用倒排索引求交集，替代逐对比较。在真实语料上结果与原逐对实现逐个相同，耗时下降约 5.6 倍。
- 聚类在分组未发生变化时不再写库。此前每 75 秒的分析循环都会重写窗口内全部 `cluster_id`，是 WAL 持续膨胀的主因。
- 热点榜的时间衰减热度改在 SQL 内计算并直接分页，不再每次请求把上千行读进内存后在 JS 里排序。
- `/api/stats` 的全表计数加 5 秒缓存，并在采集、信源变更后立即失效；界面每 18 秒的轮询不再反复全表扫描。
- 新增 `fetched_at`、`COALESCE(published_at, fetched_at)` 与 `(relevant, fetched_at)` 索引，覆盖统计、时间线排序与保留清理的查询路径。

### Fixed

- 修复信息流加载期间切换领域或分类被静默丢弃的问题：整表重载改为以最后一次请求为准，过期响应不再覆盖界面；分页追加仍保持串行。
- 修复检索词中的 `%` 与 `_` 被当作 LIKE 通配符的问题，短词降级检索现在按字面匹配。
- 右侧热度栏同样丢弃过期响应，快速切换领域不会再看到上一个领域的榜单。

### Tests

- 新增倒排索引与逐对比较在多阈值、多领域、含近重复语料下的逐对等价性回归。
- 新增保留清理的 FTS 同步、事件簇收敛与主条改选、日报独立保留期、单轮删除上限等回归。
- 新增失败退避的时长推导、到期判定、健康度分级，以及采集层「失败→跳过→到期重试→成功复位」与强制重试的集成回归。
- 新增热度排序、热度窗口回退、LIKE 转义、数据维护接口与保留天数校验的 HTTP 级回归。

## [0.0.2] - 2026-07-23

### Added

- 使用独立于随机本地端口的 `ui-preferences.json` 记住全部有意义的界面选择：主题、最后视图、领域、动态分类、日报日期、常用网址分类与星标，以及实时更新开关。
- 通过版本化白名单、原子写入和损坏回退，使上述选择在完全退出并重新打开桌面应用后仍能恢复。

### Fixed

- 修复 Electron utility process 的 `MessageEvent.data` 凭据确认未被识别，导致 DeepSeek API Key 虽已加密写入却在界面中显示保存失败、运行时无法调用的问题。
- 将 AI 设置更新、加密凭据保存和普通设置落盘串行为完整事务，避免并发更新失败回滚覆盖较新的成功设置。

### Tests

- 新增两次真实 Electron 冷启动回归测试，强制第二次启动使用不同随机端口，并通过本地模拟 DeepSeek 服务验证加密 Key 恢复和模型调用。
- 新增界面偏好存储、IPC 边界、渲染器恢复、凭据确认及设置事务的单元回归测试。

## [0.0.1] - 2026-07-21

### Added

- Windows x64 桌面安装包、自动更新元数据与本地 SQLite 情报库。
- 低空经济、商业航天信息流、全文检索、聚类、日报与信源管理。
- 可选 AI 五维评分、系统加密凭据存储与“云幄 · 常用网址”视图。

### Security

- 随机回环端口、启动令牌、严格 Host/Origin 校验与请求体限制。
- Electron sandbox、权限默认拒绝、CSP、安全外链与渲染崩溃恢复。
- 原子设置写入、SQLite 完整性检查、旧数据一致性迁移与优雅退出。
- 白名单打包、ASAR 审计、生产依赖审计、SBOM 与 SHA-256 发布门禁。
