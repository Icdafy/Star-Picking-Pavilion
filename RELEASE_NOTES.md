# 摘星阁 v0.0.3

本版本面向「装上就一直开着」的长期运行场景。此前摘星阁越用越沉：本地库只进不出，后台每 75 秒的聚类会阻塞事件循环并持续写入，失效信源每轮都在空转。这一版把这三件事逐一解决。

## 亮点

- **本地库不再无限膨胀。** 情报按可配置天数保留（默认 180 天），判为无关的噪声用更短的独立天数（默认 21 天）。到期条目连同 FTS5 全文索引行一起清除，事件簇随之收敛，清理后截断 WAL。每天日报后 20 分钟自动执行。
- **设置页可以看见并管理本地库。** 新增「数据维护」卡片，显示库内情报数、实际占用空间与待清理条数，可调整两档保留天数，也可立即清理。
- **失效信源不再空转。** 连续失败的信源按指数拉长重试间隔（一个采集周期起，上限 6 小时），成功即刻复位。信源页显示暂停状态与连续失败次数，并提供「立即重试」；手动「立即采集分析」始终忽略退避，全量重试一遍。
- **后台不再周期性卡顿。** 事件聚类改用倒排索引求交集，在真实语料上结果与原逐对实现逐个相同，耗时下降约 5.6 倍；分组没有变化时完全不写库。
- **界面响应更实在。** 热点榜的时间衰减热度改在 SQL 内计算并直接分页；统计计数加了短缓存并在采集、信源变更后立即失效。
- **修复加载期间切换筛选被吞掉的问题。** 信息流重载以最后一次请求为准，过期响应不再覆盖界面。检索词里的 `%` 和 `_` 现在按字面匹配。

## 升级说明

数据库结构变更是幂等补列与补索引，首次启动自动完成，不会丢失既有文章、信源或日报。首次自动清理在启动约 90 秒后执行；如果库里积压了大量过期内容，清理会分批进行，每轮有删除上限，不会长时间占用数据库。

保留天数可随时在「设置 → 数据维护」调整，最短 7 天。若希望保留全部历史，把情报保留天数调到上限 3650 天即可。

## 安全与隐私

本版本未改变凭据与网络策略。API Key 仍使用 Windows 当前用户的系统加密能力保存于 `credentials.v1.json`，不会写入 `settings.json` 或界面偏好文件。新增的 `/api/maintenance` 系列接口与既有接口一样，只监听 `127.0.0.1` 的随机端口并受启动令牌保护。数据保留清理完全在本机进行，不上传任何内容。

## 安装提示

本版本未签名。Windows SmartScreen 可能提示未知发布者；请先使用随 Release 提供的 `SHA256SUMS.txt` 核验安装包，再决定是否运行。

系统要求：Windows 10/11 x64。卸载默认保留 `%APPDATA%\摘星阁` 中的数据。

启用 AI 后，文章标题、摘要等待分析文本会发送给用户配置的模型服务。详情见 README 的隐私说明。
